## 기술 스택

| 항목 | 버전 | 비고 |
|------|------|------|
| **Vitest** | 4.1.4 | Vite 기반 테스트 러너. Jest 호환 API |
| **@vitest/ui** | 4.1.4 | 브라우저에서 테스트 결과를 시각화하는 UI |

### Jest 대신 Vitest를 선택한 이유
- Vite의 설정(path alias `@/`, TypeScript, 플러그인)을 그대로 재사용 — 별도 Babel/Jest 변환 설정 불필요
- ESM 모듈 방식과 완전 호환
- 속도가 빠름 (esbuild 기반 변환)

## 설정

### vite.config.ts

```ts
/// <reference types="vitest" />
import { defineConfig } from 'vite'

export default defineConfig({
  // ...기존 설정...
  test: {
    globals: true,       // describe/it/expect를 import 없이 전역 사용
    environment: 'node', // 순수 함수 테스트 — DOM 불필요
  },
})
```

### tsconfig.app.json

```json
"types": ["vite/client", "vitest/globals"]
```

`vitest/globals`를 추가해야 TypeScript가 `describe`, `it`, `expect` 등의 전역 타입을 인식한다.

## 테스트 대상 함수 (`src/lib/utils.ts`)

### `formatDate(dateInput, dayofweek, short, english, time_display)`

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `dateInput` | — | `string \| Date \| undefined` |
| `dayofweek` | `true` | 요일 포함 여부 |
| `short` | `true` | 단축 요일 (일 / Sun) |
| `english` | `false` | 영어 요일 |
| `time_display` | `false` | HH:mm:ss 포함 여부 |

주요 테스트 케이스:

| 입력 | 옵션 | 기대 출력 |
|------|------|-----------|
| `undefined` | 기본 | `'-'` |
| `'invalid'` | 기본 | `'Invalid Date : invalid'` |
| `'20260412'` | 기본 | `'2026-04-12 (일)'` — yyyymmdd 자동 정규화 |
| `'2026-04-12'` | 기본 | `'2026-04-12 (일)'` |
| `'2026-04-12'` | `dayofweek=false` | `'2026-04-12'` |
| `'2026-04-12'` | `short=false` | `'2026-04-12 (일요일)'` |
| `'2026-04-12'` | `english=true` | `'2026-04-12 (Sun)'` |

> `time_display` 케이스는 `new Date('yyyy-mm-dd')` 가 UTC 기준 파싱되어 로컬 타임존에 따라 시간이 달라지므로 테스트에서 제외.

### `getDayOfWeek(dateStr, short, english)`

| 입력 | 옵션 | 기대 출력 |
|------|------|-----------|
| `'20260412'` | 기본 | `'일요일'` |
| `'2026-04-12'` | `short=true` | `'일'` |
| `'2026-04-12'` | `english=true` | `'Sunday'` |
| `'2026-04-12'` | `english=true, short=true` | `'Sun'` |

> `toLocaleDateString` 결과는 Node.js 빌드에 번들된 ICU 데이터에 의존한다. 한국어 locale 테스트가 실패하는 환경이라면 영어 locale 케이스만 단언하거나 `vi.spyOn`으로 격리한다.

## 새 테스트 추가 방법

1. 테스트할 파일 옆에 `파일명.test.ts` 생성
2. `describe` → 함수 단위, `it` → 케이스 단위로 구성
3. 순수 함수(입력 → 출력)는 `expect(fn(input)).toBe(expected)` 패턴 사용
4. `npm run test:watch`로 개발하면 저장 즉시 결과 확인 가능

```ts
import { describe, it, expect } from 'vitest'
import { myFunction } from './myModule'

describe('myFunction', () => {
  it('정상 입력', () => {
    expect(myFunction('foo')).toBe('expected')
  })
})
```
