# 구현 내용

## diary

###  처리 흐름

####  POST/PUT /diary (multipart/form-data)
  - diary part: JSON (Content-Type: application/json)
  - files part: 첨부파일 (optional)

#### 에디터 이미지 (register/modify 공통):
  1. content HTML에서 src="data:image/TYPE;base64,..." 패턴 추출
  2. editor-images-dir/yyyy/MM/dd/UUID.ext 로 디스크 저장
  3. files 테이블 insert → fileId 획득
  4. diary 저장 → id 획득
  5. file_match insert (file_type='editor-image')
  6. content의 base64를 file.image.base-url/yyyy/MM/dd/UUID.ext URL로 치환 후 저장

#### 첨부파일:
  1. attach-files-dir/yyyy/MM/dd/UUID.ext 로 디스크 저장
  2. files 테이블 insert
  3. file_match insert (file_type='attachment')

###  추가된 삭제 처리                                                                                         
                                                                                                                                                                                                                                       
  DiaryDto → deletedAttachmentIds: List<Long> 필드 추가                                                                                                                                                                                
                                                                                                                                                                                                                                       
  FileMapper + XML → 3개 쿼리 추가                                                                                                                                                                                                     
  - selectFilesByIds — 물리 파일 경로 조회                                                                                                                                                                                             
  - deleteFileMatchByFileIds — file_match 삭제 (IN 절)
  - deleteFilesByIds — files 레코드 삭제 (IN 절)

  FileUploadService.deleteAttachments(fileIds) → 순서대로 처리:
  1. files 경로 조회 → 2. file_match 삭제 → 3. files 삭제 → 4. 물리 파일 삭제
  (물리 파일 삭제 실패 시 경고 로그만 남기고 DB 트랜잭션은 유지)

  DiaryServiceImpl.modify() → 가장 먼저 실행:
  fileUploadService.deleteAttachments(diaryDto.getDeletedAttachmentIds());
